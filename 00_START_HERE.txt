KEY FINDINGS
════════════════════════════════════════════════════════════════════════════

1. NO UNIVERSAL LISP STANDARD EXISTS
   Different languages use different names:
     Janet: for
     Clojure: for (lazy) + doseq (eager)
     Scheme: for-each
     Common Lisp: dolist, loop

2. ELLE ALIGNS WITH MODERN LISPS
   Janet (most modern) uses "for" → Elle uses "for" ✓
   Both are actively developed modern Lisps

3. SEMANTICS ARE CLEAR
   Elle's `for` = eager side-effect iteration
   Matches: Janet's `for`, Clojure's `doseq`, CL's `dolist`
   Different from: Clojure's `for` (lazy)

4. SYNTAX IS OPTIMAL
   Elle: (for i [1 2 3] (print i))
   Janet: (for i in [1 2 3] (print i))
   Both are the most concise among compared languages

5. GROWTH PATH IS CLEAR
   If lazy sequences added later:
     Current: (for i [1 2 3] (print i))
     Future:  (for [i [1 2 3]] (+ i 1))    [lazy]
              (doseq [i [1 2 3]] ...)      [eager]
   Follows Clojure's proven model naturally

Q: What does Janet use?
A: Janet uses `for` for eager iteration (same as Elle!)
   See: lisp_iteration_research.md

Q: What does Clojure use?
A: Clojure uses `for` (lazy) and `doseq` (eager)
   Elle matches `doseq` semantics (but uses `for` name)
   See: lisp_iteration_research.md

Q: Is there a Lisp standard I should follow?
A: No! Different Lisps use different names.
   Naming choice is a design decision, not violation of standard.
   See: INDEX.md "Research Findings"

Q: Will users be confused by the naming?
A: No. Users expect "for" loops and the semantics are clear.
   Janet users will find it familiar.
   Clojure users will understand it's eager (like doseq).
   See: elle_for_recommendation.md "What Users Will Find"

Q: How does this compare to other Lisps?
A: Elle's `for` is:
   ✓ Nearly identical to Janet's
   ✓ Semantically equivalent to Clojure's `doseq`
   ✓ Similar purpose to Common Lisp's `dolist`
   ✓ Similar purpose to Scheme's `for-each`
   See: COMPARISON_TABLE.txt

Q: What if we want to add lazy sequences later?
A: Your naming now preserves a natural evolution:
   - Keep `for` as eager now
