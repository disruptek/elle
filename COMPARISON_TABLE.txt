LOOP: Print items from list [1 2 3]

╭─ Janet ────────────────────────────────╮
│ (for i in [1 2 3]                      │
│   (print i))                           │
╰────────────────────────────────────────╯

╭─ Elle (RECOMMENDED) ───────────────────╮
│ (for i [1 2 3]                         │
│   (print i))          ← Nearly identical
╰────────────────────────────────────────╯

╭─ Clojure (doseq) ──────────────────────╮
│ (doseq [i [1 2 3]]                     │
│   (println i))                         │
╰────────────────────────────────────────╯

╭─ Scheme ───────────────────────────────╮
│ (for-each (lambda (i) (display i))     │
│           '(1 2 3))                    │
╰────────────────────────────────────────╯

╭─ Common Lisp ──────────────────────────╮
│ (dolist (i '(1 2 3))                   │
│   (print i))                           │
╰────────────────────────────────────────╯

OBSERVATIONS:
  • Elle's syntax is nearly identical to Janet's
  • Elle is more concise than Clojure, Common Lisp
  • Scheme requires lambda/procedure, more verbose
  • Elle and Janet are the winners for clarity

USES "for" AS PRIMARY NAME?
  ✓ Janet (modern Lisp)          YES - for eager iteration
  ✓ Elle (modern Lisp)           YES - for eager iteration (MATCHES JANET!)
  ~ Clojure (modern Lisp)        YES, but only for LAZY variant (different semantics!)
  ✗ Scheme (traditional Lisp)    NO - uses for-each
  ✗ Common Lisp (traditional)    NO - uses dolist, loop

INSIGHT: "for" is used by MODERN lisps, not TRADITIONAL lisps
         But Clojure's "for" means something different (lazy)!

Elle's choice aligns with JANET, the most modern active Lisp.
